## Critical

### 1. Auth tokens stored in plain SQLite (`UserEntity`)
`tokenEnc` / `refreshEnc` sit in `users` table. Name implies encryption but there is no encryption at rest. SQLite DB is readable on rooted devices.  
**Fix:** Move tokens to `EncryptedSharedPreferences` (Jetpack Security Crypto). Keep only non-sensitive profile fields in Room.

### 2. Enum converters crash on unknown values (`AppConverters`)
`enumValueOf<Genre>(it)`, `enumValueOf<DownloadState>(it)`, `enumValueOf<RatingTargetType>(it)` throw `IllegalArgumentException` if DB contains a value not in current enum (e.g. after removing a member or receiving a future server value).  
**Fix:**
```kotlin
fun toGenre(name: String?): Genre? = name?.let {
    Genre.entries.find { e -> e.name == name }
}
```
Apply same pattern to all three `to*` converters.

### 3. No migration strategy in `DatabaseModule`
`Room.databaseBuilder(...).build()` with no `.addMigrations()` and no `.fallbackToDestructiveMigration()`.  
On any schema version bump the app crashes with `IllegalStateException` at startup.  
**Fix (dev):** Add `.fallbackToDestructiveMigration()` now.  
**Fix (prod):** Write explicit `Migration` objects and register them.

### 4. `exportSchema = false` blocks migration safety net
Without schema exports Room cannot validate that your `Migration` SQL matches the actual expected schema.  
**Fix:** Set `exportSchema = true`, add `room.schemaLocation` to `build.gradle.kts`, commit schema JSONs.

## High

### 5. No `@ForeignKey` constraints
`DownloadEntity.trackId`, `PlaylistTrackCrossRef.playlistId/trackId`, `SubtitleLineEntity` (assumed), `RatingEntity.targetId` — none declare Room FK constraints.  
Orphaned rows accumulate silently when parent rows are deleted.  
**Fix:** Add `@ForeignKey` with `onDelete = CASCADE` where appropriate.

### 6. `PlaylistDao.getTracksForPlaylist` returns cross-ref rows, not tracks
```kotlin
fun getTracksForPlaylist(playlistId: String): Flow<List<PlaylistTrackCrossRef>>
```
Callers get join-table rows, not `TrackEntity` objects. To render a playlist you need a second query.  
**Fix:** Add a `@Transaction` + `@Relation` query or a raw JOIN returning `TrackEntity`.

### 7. `PlaylistTrackCrossRef` missing index on `trackId`
Composite PK covers `(playlistId, trackId)`. Querying "which playlists contain this track?" does a full-table scan.  
**Fix:**
```kotlin
@Entity(
    tableName = "playlist_track_cross_refs",
    primaryKeys = ["playlistId", "trackId"],
    indices = [Index("trackId")]
)
```

### 8. `ProviderEntity.manifestUrl` not validated as HTTPS
`systemPatterns.md` mandates HTTPS-only for provider manifests. Nothing enforces this at insert time.  
**Fix:** Validate in the repository layer before calling `providerDao.insert()`.

## Medium

### 9. `PendingOpEntity.type` is untyped `String`
No compile-time guard against typos or invalid op types.  
**Fix:** Replace with a `PendingOpType` enum (or sealed class) + `@TypeConverter`.

### 10. `ProviderEntity.capabilities` is raw `String`
Likely JSON or CSV. Opaque to queries and type-unsafe.  
**Fix:** Use `List<String>` + a `@TypeConverter` that serializes to JSON, or a separate `provider_capabilities` table.

### 11. `TrackEntity.streamUrl` / `artworkUrl` — no expiry tracking
If these are presigned MinIO URLs they expire. App has no way to know when to refresh.  
**Fix:** Add `streamUrlExpiresAt: Instant?` field (nullable for non-expiring sources).

### 12. `RatingEntity.value: Int` — no bounds
Nothing prevents `value = -999`. Domain constraint (e.g. 1–5) should be enforced.  
**Fix:** Validate in repository or use a `RatingValue` value class with `init { require(value in 1..5) }`.

### 13. `DatabaseModule` — no `.fallbackToDestructiveMigration()` for debug builds
Already covered in #3 but specifically: during development without this, every schema change requires a manual uninstall.

## Low / Housekeeping

### 14. Zero DAO test coverage (92 test gaps)
All DAOs, entities, and converters are untested. Room's in-memory driver makes DAO tests trivial.  
**Minimum viable:** `AppConvertersTest` (pure unit), one `TrackDaoTest` with `@RunWith(AndroidJUnit4::class)` + in-memory `Room.inMemoryDatabaseBuilder`.

### 15. `MainActivity` still has boilerplate `Greeting` composable
Real nav host not wired yet. Track in `activeContext.md` as a known gap.

### 16. Module structure is package-only
`progress.md` notes this: `:core:*` and `:feature:*` Gradle modules not split yet. Low urgency but the longer it waits the more expensive the refactor.
